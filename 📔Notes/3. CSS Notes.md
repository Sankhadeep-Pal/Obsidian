## [[css.pdf|What is CSS?]]
**CSS (Cascading Style Sheets)** is a fundamental technology in web development used to control the appearance and layout of web pages. It works alongside **HTML** (which provides structure) and **JavaScript** (which adds interactivity).

### What does CSS do?

- Styles **text** (fonts, colors, sizes)

- Adjusts **layouts** (positioning, spacing, alignment)

- Handles **responsive design** (adapts to screen sizes)

- Adds **visual effects** (animations, transitions, shadows)

### Why CSS Matters:

- Separates content from presentation

- Enhances user experience

- Supports maintainable and scale-able web design

- Enables accessibility and mobile optimization

## How to Use CSS?
We can apply **CSS** to **HTML Docs** in **3 different ways**.

#### 1. **Inline CSS**

Directly in HTML elements:

- Apply only on that particular element.

- Not practical at all

```html
<p style="color: blue;">Hello World!</p>
```

#### 2. **Internal CSS**

Within a `<style>` tag in the HTML `<head>` tag:

- Apply only on that file.

- Have limited practical use case.

```html
<head>
	<style>
        p { 
        	color: red; 
        }
	</style>
</head>
```

#### 3. **External CSS**

In a separate `.css` file, linked in HTML:

- Apply to all those HTML which are link to this.

- This method is used in maximum cases.

Inside the `<head>` tag in HTML
```html
<head>
	<link rel="stylesheet" href="styles.css">
</head>
```

In the separate `.css` file
```css
p {   
	color: green; 
}
```

## CSS Color System
CSS provides several ways to define colors when styling HTML elements. Understanding these systems helps you create visually appealing and accessible designs. Below are the most commonly used color formats in CSS.

### 1. **Named Colors**
CSS includes around 140 predefined **named colors** that are easy to use and remember. These are built-in keywords that correspond to specific colors.

#### ‚úÖ Example Usage:

```css
p {   
	color: red; 
}
```

#### üìã Common Named Colors:

- `red`
- `blue`
- `green`
- `black`
- `white`
- `gray`
- `gold`
- `darkcyan`
- `lightcoral`

#### ‚úÖ Best for:

- Quick styling

- Simple designs

- When precise color matching isn't needed

### 2. **[[Hexadecimal]] Colors**
**Hex codes** represent colors using hexadecimal values (base-16). They begin with `#` and contain either **3** or **6** digits:

#### ‚úÖ Syntax:

```
#RRGGBB   /* Red, Green, Blue values in hex (00 to FF) */ 
#RGB      /* Shorthand for #RRGGBB */
```

#### ‚úÖ Example Usage:

```css
h1 {   
	color: #ff0000;   /* Red */ 
} 
div {   
	background-color: #0f0;   /* Shorthand for #00ff00 (Green) */ 
}
```

#### ‚úÖ Best for:

- Consistent, precise color design

- Web-safe and design tool compatibility

### 3. **RGB Colors**
**RGB** stands for **Red, Green, Blue**. This format uses decimal values between `0` and `255` for each color channel.

![[Screenshot From 2025-06-18 11-51-52.png]]
#### ‚úÖ Syntax:

```
rgb(red, green, blue)
```

Each parameter can be:

- A number between `0` and `255`

- A percentage (optional in some contexts)

#### ‚úÖ Example Usage:

```css
p {   
	color: rgb(255, 0, 0);   /* Red */ 
} 
section {   
	background-color: rgb(0, 128, 255);   /* Light blue */ 
}
```

#### ‚úÖ Best for:

- Programmatic control (like with JavaScript)
 
- Consistent visual results

### 4. **RGBA Colors**
**RGBA** is an extension of RGB that adds **Alpha** transparency (opacity level). The alpha value ranges from `0` (fully transparent) to `1` (fully opaque).

#### ‚úÖ Syntax:

```
rgba(red, green, blue, alpha)
```

#### ‚úÖ Example Usage:

```css
.box {
	background-color: rgba(255, 0, 0, 0.5);   /* Semi-transparent red */ 
}
```

#### ‚úÖ Best for:

- Creating layered effects

- Smooth transparency in UI elements
 
- Hover effects, shadows, and overlays

### 5. Opacity
In **CSS**, the `opacity` property controls how transparent an element is.
#### ‚úÖ Syntax:

```css
selector {   
	opacity: value; 
}
```
- **`value`** is a number between **0** (completely transparent) and **1** (completely opaque)

#### ‚úÖ Important Notes:

- `opacity` affects the entire element **and all of its children**.

- If you want only the background or a specific part to be transparent (not the text inside, for example), use `rgba()` for color instead.

## Unit in CSS
In CSS, **units** define **lengths**, such as width, height, margins, padding, font size, etc.
There are two main types of units:

### 1. **Absolute Units**
These are **fixed** units that do **not change** based on screen size or parent elements. They‚Äôre best for print or situations where exact sizing is required.

|Unit|Description|Example|
|---|---|---|
|`px`|Pixels (1in = **96px**)|`font-size: 16px;`|
|`pt`|Points (1 pt = 1/72 in)|`margin: 12pt;`|
|`cm`|Centimeters (1in = 2.54cm)|`width: 5cm;`|
|`mm`|Millimeters (10mm = 1cm)|`height: 30mm;`|
|`in`|Inches|`width: 2in;`|

‚úÖ **Use When:** You want fixed dimensions regardless of screen size.

### 2. **Relative Units**
These **scale** based on the context ‚Äî like the font size of the parent or the size of the viewport. They're **more flexible** and ideal for **responsive design**.

#### üìê Font-Relative Units:

|Unit|Description|Example|
|---|---|---|
|`em`|Relative to parent element‚Äôs font size|`padding: 2em;`|
|`rem`|Relative to the **root** element‚Äôs font size|`font-size: 1.5rem;`|

- `1em` = 100% of the parent font size

- `1rem` = 100% of the root (`html`) font size

#### üì± Viewport-Relative Units:

|Unit|Description|Example|
|---|---|---|
|`vw`|1% of the **viewport width**|`width: 50vw;`|
|`vh`|1% of the **viewport height**|`height: 100vh;`|
|`vmin`|1% of the **smaller** viewport side|`font-size: 5vmin;`|
|`vmax`|1% of the **larger** viewport side|`margin: 2vmax;`|
|`%`|Relative to the **parent element‚Äôs size**|`width: 80%;`|

- `%` is **context-dependent**:
    
    - For width/height ‚Üí based on parent‚Äôs width/height
    
    - For padding/margin ‚Üí based on parent‚Äôs width
 
### Special Unit: `fr` (for Grid)

- `fr` = "fractional unit" used in **CSS Grid Layout**

```css
grid-template-columns: 1fr 2fr;  /* 1 part vs 2 parts */
```

### Best Practices

- Use `rem` or `em` for font sizes for better accessibility.

- Use `px` for borders, shadows, and fine control.

- Use `vw`/`vh` for responsive full-page sections.

- Avoid mixing too many unit types in one block.

## Selector in CSS
In CSS, **selectors** are patterns used to **target HTML elements** that you want to style. They tell the browser **which elements** your CSS rules should apply to.

![[Screenshot From 2025-06-19 21-54-56.png]]
### Common Types of Selectors

#### 1. Universal Selector
Selects **all** elements:

```css
* {   
	margin: 0; 
}
```

#### 2. Type Selector
Selects all elements of a given tag:

```css
h1 {   
	font-size: 2rem;
}
```

#### 3. Class Selector
Targets elements with a specific class:

```css
.card {   
	border: 1px solid #ccc; 
}
```

```html
<div class="card"></div>
```

#### 4. ID Selector
Targets an element with a specific ID:

```css
#main-header {   
	background-color: black; 
}
```

```html
<h1 id="main-header">Title</h1>
```

#### 5. Group Selector
Targets multiple selectors at once:

```css
h1, 
h2, 
h3 {   
	font-family: Arial; 
}
```

#### 6. Descendant Selector
Targets elements **inside** other elements:

```css
nav a {   
	color: green; 
}
```
<mark style="background: #FFF3A3A6;"><b>Note:</b> This selector can jump one or, multiple generation to target child, grandchild or, farther lower generation unlike <i>Child Combinator</i> which can target only direct children.</mark>
#### 7. Child Combinator (`>`)
Targets **direct children** only:

```css
ul > li {   
	list-style: square; 
}
```

#### 8. Adjacent Sibling Combinator (`+`)
Selects an element **immediately after** another:

```css
h1 + p {   
	margin-top: 0; 
}
```

#### 9. [Attribute Selector](https://developer.mozilla.org/en-US/docs/Web/CSS/Attribute_selectors)
Targets elements by their attributes:

```css
input[type="text"] {   
	border: 1px solid #999; 
}
```

### Pseudo Class & Pseudo Element in CSS

#### [Pseudo-Classes (`:`)](https://developer.mozilla.org/en-US/docs/Web/CSS/Pseudo-classes)
A **pseudo-class** is used to define the **special state** of an element. It **doesn't add new content**, but styles the element **based on conditions** like hover, focus, or position in the DOM.

##### üìã **Syntax:**

```css
selector:pseudo-class {   
	/* style rules */
}
```

##### üî∂ **Common Pseudo-Classes:**

|Pseudo-Class|Description|Example|
|---|---|---|
|`:hover`|When the user hovers over an element|`a:hover { color: red; }`|
|`:focus`|When an element (like input) is focused|`input:focus { border-color: blue; }`|
|`:checked`|When a checkbox/radio is selected|`input:checked + label { color: green; }`|
|`:nth-child(n)`|Selects the nth child|`li:nth-child(2) { font-weight: bold; }`|
|`:first-child`|Selects the first child of a parent|`p:first-child { margin-top: 0; }`|
|`:last-child`|Selects the last child of a parent|`li:last-child { color: gray; }`|
|`:not(selector)`|Excludes elements matching the selector|`p:not(.highlight) { color: black; }`|

#### [Pseudo-Elements (`::`)](https://developer.mozilla.org/en-US/docs/Web/CSS/Pseudo-elements)
A **pseudo-element** allows you to **style specific parts** of an element or **insert virtual content** before or after an element ‚Äî without needing to change the HTML.

##### üìã **Syntax:**

```css
selector::pseudo-element {   
	/* style rules */ 
}
```
<mark style="background: #FFF3A3A6;"><b>Note:</b>Modern CSS uses <i>double colons (`::`)</i> to distinguish pseudo-elements from pseudo-classes, but older browsers still support single colons.</mark>

##### üî∏ **Common Pseudo-Elements:**

|Pseudo-Element|Description|Example|
|---|---|---|
|`::before`|Inserts content **before** the element|`h1::before { content: "üëâ "; }`|
|`::after`|Inserts content **after** the element|`a::after { content: " üîó"; }`|
|`::first-letter`|Styles the **first letter** of text|`p::first-letter { font-size: 200%; }`|
|`::first-line`|Styles the **first line** of a paragraph|`p::first-line { color: navy; }`|
|`::selection`|Styles text that is **highlighted by the user**|`::selection { background: yellow; }`|

## Cascading & Specificity in CSS
### What is **Cascading** in CSS?
The **‚ÄúC‚Äù in CSS** stands for **Cascading**. It describes how **multiple CSS rules are applied in a specific order** of priority when they affect the same element.

#### üìö Cascade Determines Priority Based On:

1. **Origin**: Browser default, user styles, author styles

2. **Importance**: `!important` overrides everything (use carefully)

3. **Specificity**: How targeted the rule is

4. **Source Order**: Last rule wins if specificity is equal

#### üî∂Example:

```css
/* Rule 1 */
p {
  color: blue;
}

/* Rule 2 */
#main p {
  color: red;
}

/* Rule 3 */
p {
  color: green !important;
}
```
<mark style="background: #ADCCFFA6;">Rule 3 wins because it uses `!important`, even though Rule 2 is more specific than Rule 1.</mark>

### 2. What is **Specificity**?
Specificity is a score that determines which rule wins when multiple rules apply to the same element. The **higher the specificity**, the more "weight" that selector has.

#### üßÆ Specificity Formula:
CSS calculates specificity as a **four-part value**:

$$
a (inline styles), b (id), c (.class, [attr], :pseudo-class), d (element, ::pseudo-element)
$$

#### üßæ Examples (from lowest to highest):

|Selector|Specificity|Notes|
|---|---|---|
|`p`|0,0,0,1|Element selector|
|`.text`|0,0,1,0|Class selector|
|`#main`|0,1,0,0|ID selector|
|`div#main .text`|0,1,1,1|Combination of ID + class + element|
|`style="..."`|1,0,0,0|Inline style always wins (unless `!important`)|

### `!important` Overrides Everything

If you write:

```css
p {
  color: red !important;
}
```

It will override even inline styles and highly specific selectors ‚Äî unless those also have `!important`.

==**Disclaimer:** ‚ö†Ô∏è *Use '! Important' sparingly* ‚Äì it breaks the natural cascade and can make debugging harder.==

### Best Practices

- Prefer **specific classes** over IDs in CSS (more reusable and easier to override).

- Avoid relying heavily on `!important`.
 
- Understand how rules stack and resolve conflicts.

## Inheritance in CSS
**Inheritance** means that some CSS property values are **passed down** (inherited) from a parent element to its children.

Just like in families, **child elements can inherit traits (styles)** from their parents ‚Äî but **not all traits** are passed down automatically.

### üëë How Inheritance Works
When you apply a property to a parent element, **only certain properties** will automatically be inherited by its children ‚Äî usually those related to **text and appearance**.

### üìÉ Commonly Inherited Properties:

- `color`
- `font-family`
- `font-size`
- `font-style`
- `font-variant`
- `font-weight`
- `letter-spacing`
- `line-height`
- `text-align`
- `visibility`
- `white-space`
- `word-spacing`

### üî∂ Not Inherited by Default:

- `margin`, `padding`, `border`
- `width`, `height`
- `background`, `box-shadow`
- `position`, `display`
- `flex`, `grid`, `z-index`

### üî∂ Forcing Inheritance
If a property is **not inherited by default**, you can make it inherit manually:

```css
div {
  background-color: inherit;
}
```

This tells the child `<div>` to **explicitly copy** the background color from its parent.

### ‚úÖ Example:

```html
<div class="parent">
  Parent Text
  <p class="child">Child Text</p>
</div>
```

```css
.parent {
  color: darkblue;
  font-family: Arial;
}

.child {
  /* Inherits both 'color' and 'font-family' */
}
```
Result: Both the parent and child text will appear in **dark blue Arial**.

## Box Model in CSS
The **Box Model** in CSS is a fundamental concept that describes how elements on a web page are structured and how spacing and sizing work. Every HTML element is essentially a rectangular box, and the CSS box model defines how the size of that box is calculated.

### Parts of the Box Model

1. **Content**
    
    - The actual content of the box (text, image, etc.)
    
    - Controlled by properties like `width` and `height`.

2. **Padding**
    
    - Space between the content and the border.
    
    - Adds spacing _inside_ the element.
    
    - Controlled by `padding`, `padding-top`, `padding-right`, etc.

3. **Border**
    
    - Surrounds the padding (if any) and content.
    
    - Controlled by `border`, `border-width`, `border-style`, etc.

4. **Margin**
    
    - Space _outside_ the border, separating the element from others.
    
    - Controlled by `margin`, `margin-top`, `margin-right`, etc.

### Visual Representation

![[Screenshot From 2025-06-22 11-57-55.png]]
### Box Model Properties

#### 1. `Height` & `Width` property:
In the **CSS box model**, the `height` and `width` properties define the size of the **content area** of an element ‚Äî **not including** padding, border, or margin **unless** otherwise specified.

##### üìê **How `height` and  `width` Work:**

```css
.box {
  width: 200px;
  height: 100px;
}
```

This sets the **content area** to `200px` wide and `100px` tall. Padding, border, and margin are **added on top** of that.

##### üßÆ **Total Element Size (default box model):**
By default (`box-sizing: content-box`), total size:

```
Total width  = width + padding-left + padding-right + border-left + border-right
Total height = height + padding-top + padding-bottom + border-top + border-bottom
```

##### ‚úÖ **Using `box-sizing: border-box`**
To make the `width` and `height` include padding and border:

```css
.box {
  width: 200px;
  height: 100px;
  padding: 20px;
  border: 5px solid black;
  box-sizing: border-box;
}
```

Now, the total size stays at `200√ó100px`, and the content shrinks to make room for padding and border.

#### Border Property:
In the **CSS box model**, the `border` is the layer **between the padding and the margin**. It visually wraps the content and padding, and it **adds to the total size of the element** (unless `box-sizing: border-box` is used).

You can define borders using individual properties or shorthand.

##### üß± **Basic Border Properties:**

```css
border-width: 2px;      /* thickness */
border-style: solid;    /* type: solid, dashed, dotted, etc. */
border-color: black;    /* color */
```

Or use shorthand:

```css
border: 2px solid black;
```

##### üî∂ **Side-Specific Borders**
You can target each side independently:

```css
border-top: 1px solid red;
border-right: 2px dashed green;
border-bottom: 3px dotted blue;
border-left: 4px double orange;
```

##### üé® **Border Variants**

- `border-style` options:  
    `none`, `solid`, `dashed`, `dotted`, `double`, `groove`, `ridge`, `inset`, `outset`

- `border-radius`: adds rounded corners

```css
border-radius: 8px;
```
There are also corner specific `border-radius`.

- For a **perfect circle**:
    
    - The element must have equal `width` and `height`.
    
    - Use `border-radius: 50%`.

```css
.circle {   
	width: 100px;   
	height: 100px;   
	border-radius: 50%;   
	background: red; 
}
```

- For an **oval**:

```css
oval {   
	width: 200px;   
	height: 100px;   
	border-radius: 50%;   
	background: blue; 
}
```

- For an **Ellipse**:

```css
border-radius: 50px 25px;
```

#### Padding property:
The `padding` property in CSS controls the **space between the content of an element and its border**. It‚Äôs **inside** the border, unlike `margin` which is on the **outside**.

##### ‚úÖ **Basic Syntax**

```css
selector {
  padding: 20px;
}
```
This adds `20px` **of space on all four sides** (top, right, bottom, left).

Or use shorthand:

```css
padding: top right bottom left;
```

You can define 1 to 4 values:

| Syntax                         | Meaning                                         |
| ------------------------------ | ----------------------------------------------- |
| `padding: 20px;`               | All sides: 20px                                 |
| `padding: 10px 20px;`          | Top/bottom: 10px, left/right: 20px              |
| `padding: 10px 15px 20px;`     | Top: 10px, left/right: 15px, bottom: 20px       |
| `padding: 5px 10px 15px 20px;` | Top: 5px, Right: 10px, Bottom: 15px, Left: 20px |

##### üî∂ **Individual Sides**

```css
padding-top: 10px;
padding-right: 20px;
padding-bottom: 15px;
padding-left: 5px;
```

#### Margin property:
The `margin` property in CSS controls the **space _outside_ an element**, between its border and the elements around it. It‚Äôs part of the **CSS box model**, just like `padding`, `border`, and `width/height`.

##### ‚úÖ **Basic Syntax**

```css
selector {
  margin: 20px;
}
```
This applies a `20px ` **margin on all four sides**: top, right, bottom, and left.

Or use shorthand:

```css
margin: top right bottom left;
```

You can use 1 to 4 values:

|Syntax|Result|
|---|---|
|`margin: 20px;`|All four sides = 20px|
|`margin: 10px 20px;`|Top/bottom = 10px, Left/right = 20px|
|`margin: 10px 15px 20px;`|Top = 10px, Left/right = 15px, Bottom = 20px|
|`margin: 5px 10px 15px 20px;`|Top, Right, Bottom, Left (clockwise order)|

##### üî∂ **Individual Sides**

```css
margin-top: 10px;
margin-right: 15px;
margin-bottom: 10px;
margin-left: 5px;
```

##### üîÅ **Auto Margin**
You can center a block element horizontally using `auto`:

```css
margin: 0 auto;
```
This sets top/bottom to `0` and **left/right to auto**, which centers the element if it has a fixed `width`.

##### üß± **Box Model Context**

- `margin` is **outside the border**.

- It does **not affect content size**.

- Margins can **collapse** (especially vertical margins between block elements).

#### Display Property:
The `display` property in CSS controls **how an element is displayed** on the page ‚Äî it defines the element‚Äôs **layout behavior** in the document flow.

##### ‚úÖ **Common `display` Values**

|Value|Description|
|---|---|
|`block`|Element takes full width, starts on a new line|
|`inline`|Element flows with text, no line break, width/height can't be set|
|`inline-block`|Like `inline`, but you can set width/height|
|`none`|Hides the element (removes it from the layout)|
|`flex`|Enables Flexbox layout (for flexible, 1D layouts)|
|`grid`|Enables Grid layout (for 2D layouts)|
|`inline-flex`|Flex layout, behaves like inline element|
|`inline-grid`|Grid layout, behaves like inline element|

## Transition in CSS
In **CSS**, the `transition` property allows you to smoothly animate changes to CSS properties over a set duration.

### ‚úÖ Basic Syntax:

```css
selector {
  transition: property duration timing-function delay;
}
```

- **`property`** ‚Äì the CSS property to animate (e.g., `opacity`, `background-color`, `transform`)

- **`duration`** ‚Äì how long the transition lasts (e.g., `0.5s`, `200ms`)

- **`timing-function`** _(optional)_ ‚Äì the speed curve (`ease`, `linear`, `ease-in`, `ease-out`, `ease-in-out`)

- **`delay`** _(optional)_ ‚Äì time before the animation starts

### ‚úÖ Example ‚Äì Smooth Hover Effect:

```css
.button {
  background-color: blue;
  color: white;
  transition: background-color 0.3s ease;
}

.button:hover {
  background-color: darkblue;
}
```

### üî∂ Transitioning Multiple Properties:

```css
.box {
  transition: width 0.5s, height 0.5s, background-color 0.3s;
}
```

### üìÉ Common Timing Functions:

- `ease` _(default)_ ‚Äì starts slow, speeds up, then slows down
 
- `linear` ‚Äì constant speed

- `ease-in` ‚Äì starts slow

- `ease-out` ‚Äì ends slow

- `ease-in-out` ‚Äì starts and ends slow

### üî∂ All Properties (not recommended for performance):

```css
transition: all 0.3s ease;
```

## Transform In CSS
In **CSS**, the `transform` property allows you to visually manipulate elements‚Äîlike moving, scaling, rotating, or skewing‚Äîwithout affecting the document layout.

### ‚úÖ Basic Syntax:

```css
selector {
  transform: function(value);
}
```

You can chain multiple functions like:

```css
transform: rotate(45deg) scale(1.2) translateX(50px);
```

### üìÉ Common Transform Functions:

|Function|Description|Example|
|---|---|---|
|`translateX(x)`|Moves element horizontally|`translateX(100px)`|
|`translateY(y)`|Moves element vertically|`translateY(-50%)`|
|`translate(x, y)`|Moves element in both directions|`translate(10px, 20px)`|
|`scale(s)`|Scales both width and height|`scale(1.5)`|
|`scaleX(x)`|Scales width only|`scaleX(2)`|
|`scaleY(y)`|Scales height only|`scaleY(0.5)`|
|`rotate(angle)`|Rotates element (clockwise)|`rotate(45deg)`|
|`skewX(angle)`|Skews horizontally|`skewX(15deg)`|
|`skewY(angle)`|Skews vertically|`skewY(-10deg)`|
|`matrix(...)`|Advanced 2D transforms|(rarely used manually)|

### ‚úÖ Example ‚Äì Hover with Scale and Rotate:

```css
.card {
  transition: transform 0.3s ease;
}

.card:hover {
  transform: scale(1.05) rotate(3deg);
}
```

### üî∂ Note:

- Transforms do **not** move other elements around them.

- You can **combine `transform` with `transition`** for smooth animations.

## Position properties in CSS
In **CSS**, the `position` property defines **how an element is positioned in the document**, and it determines how properties like `top`, `left`, `right`, and `bottom` affect it.

### Position Values:

#### 1. `static` (default)

- Normal flow ‚Äî no `top/left/right/bottom` effects.

```css
.element {
  position: static;
}
```

#### 2. `relative`

- Element stays in the flow **but can be offset**.

```css
.element {
  position: relative;
  top: 10px; /* moves down 10px from its normal spot */
}
```

#### 3. `absolute`

- **Removed from normal flow** and **positioned relative to the nearest positioned ancestor** (not `static`).

```css
.parent {
  position: relative;
}
.child {
  position: absolute;
  top: 0;
  left: 0;
}
```

#### 4. `fixed`

- **Removed from flow** and positioned **relative to the viewport**. Doesn't scroll with the page.

```css
.banner {
  position: fixed;
  bottom: 0;
  width: 100%;
}
```

#### 5. `sticky`

- Behaves like `relative` until a scroll threshold is reached, then behaves like `fixed`.

```css
.header {
  position: sticky;
  top: 0;
  background: white;
}
```

### Example: Absolute Inside Relative

```html
<div class="container">
  <div class="box">I'm positioned!</div>
</div>
```

```css
.container {
  position: relative;
}
.box {
  position: absolute;
  top: 10px;
  left: 20px;
}
```

## Shadows in CSS
In **CSS**, "shadow" usually refers to two main types:

1. **Text Shadow** (`text-shadow`)

2. **Box Shadow** (`box-shadow`)

### 1. `text-shadow`
Applies a shadow to text.

#### ‚úÖ Syntax:

```css
text-shadow: x-offset y-offset blur-radius color;
```

#### ‚úÖ Example:

```css
h1 {
  text-shadow: 2px 2px 5px rgba(0, 0, 0, 0.3);
}
```

- `2px 2px` ‚Üí shadow moves right and down

- `5px` ‚Üí blur radius

- `rgba(...)` ‚Üí color with transparency

### 2. `box-shadow`
Applies a shadow to the **outside (or inside)** of a box element.

#### ‚úÖ Syntax:

```css
box-shadow: x-offset y-offset blur-radius spread-radius color;
```

- `x-offset`: horizontal shift

- `y-offset`: vertical shift

- `blur-radius`: how soft the shadow edge is

- `spread-radius` _(optional)_: how far it grows/shrinks

- `inset` _(optional)_: makes the shadow appear inside the box
