## [[css.pdf|What is CSS?]]
**CSS (Cascading Style Sheets)** is a fundamental technology in web development used to control the appearance and layout of web pages. It works alongside **HTML** (which provides structure) and **JavaScript** (which adds interactivity).

### What does CSS do?

- Styles **text** (fonts, colors, sizes)

- Adjusts **layouts** (positioning, spacing, alignment)

- Handles **responsive design** (adapts to screen sizes)

- Adds **visual effects** (animations, transitions, shadows)

### Why CSS Matters:

- Separates content from presentation

- Enhances user experience

- Supports maintainable and scale-able web design

- Enables accessibility and mobile optimization

## How to Use CSS?
We can apply **CSS** to **HTML Docs** in **3 different ways**.

#### 1. **Inline CSS**

Directly in HTML elements:

- Apply only on that particular element.

- Not practical at all

```
<p style="color: blue;">Hello World!</p>
```

#### 2. **Internal CSS**

Within a `<style>` tag in the HTML `<head>` tag:

- Apply only on that file.

- Have limited practical use case.

```
<head>
	<style>
        p { 
        	color: red; 
        }
	</style>
</head>
```

#### 3. **External CSS**

In a separate `.css` file, linked in HTML:

- Apply to all those HTML which are link to this.

- This method is used in maximum cases.

Inside the `<head>` tag in HTML
```
<head>
	<link rel="stylesheet" href="styles.css">
</head>
```

In the separate `.css` file
```
p {   
	color: green; 
}
```

## üé® CSS Color System
CSS provides several ways to define colors when styling HTML elements. Understanding these systems helps you create visually appealing and accessible designs. Below are the most commonly used color formats in CSS.

### 1. **Named Colors**
CSS includes around 140 predefined **named colors** that are easy to use and remember. These are built-in keywords that correspond to specific colors.

#### ‚úÖ Example Usage:

```
p {   
	color: red; 
}
```

#### üìã Common Named Colors:

- `red`
- `blue`
- `green`
- `black`
- `white`
- `gray`
- `gold`
- `darkcyan`
- `lightcoral`

#### ‚úÖ Best for:

- Quick styling

- Simple designs

- When precise color matching isn't needed

### 2. **[[Hexadecimal]] Colors**
**Hex codes** represent colors using hexadecimal values (base-16). They begin with `#` and contain either **3** or **6** digits:

#### ‚úÖ Syntax:

```
#RRGGBB   /* Red, Green, Blue values in hex (00 to FF) */ 
#RGB      /* Shorthand for #RRGGBB */
```

#### ‚úÖ Example Usage:

```
h1 {   
	color: #ff0000;   /* Red */ 
} 
div {   
	background-color: #0f0;   /* Shorthand for #00ff00 (Green) */ 
}
```

#### ‚úÖ Best for:

- Consistent, precise color design

- Web-safe and design tool compatibility

### 3. **RGB Colors**
**RGB** stands for **Red, Green, Blue**. This format uses decimal values between `0` and `255` for each color channel.

![[Screenshot From 2025-06-18 11-51-52.png]]
#### ‚úÖ Syntax:

```
rgb(red, green, blue)
```

Each parameter can be:

- A number between `0` and `255`

- A percentage (optional in some contexts)

#### ‚úÖ Example Usage:

```
p {   
	color: rgb(255, 0, 0);   /* Red */ 
} 
section {   
	background-color: rgb(0, 128, 255);   /* Light blue */ 
}
```

#### ‚úÖ Best for:

- Programmatic control (like with JavaScript)
 
- Consistent visual results

### 4. **RGBA Colors**
**RGBA** is an extension of RGB that adds **Alpha** transparency (opacity level). The alpha value ranges from `0` (fully transparent) to `1` (fully opaque).

#### ‚úÖ Syntax:

```
rgba(red, green, blue, alpha)
```

#### ‚úÖ Example Usage:

```
.box {
	background-color: rgba(255, 0, 0, 0.5);   /* Semi-transparent red */ 
}
```

#### ‚úÖ Best for:

- Creating layered effects

- Smooth transparency in UI elements
 
- Hover effects, shadows, and overlays

## Unit in CSS
In CSS, **units** define **lengths**, such as width, height, margins, padding, font size, etc.
There are two main types of units:

### 1. **Absolute Units**
These are **fixed** units that do **not change** based on screen size or parent elements. They‚Äôre best for print or situations where exact sizing is required.

|Unit|Description|Example|
|---|---|---|
|`px`|Pixels (1in = **96px**)|`font-size: 16px;`|
|`pt`|Points (1 pt = 1/72 in)|`margin: 12pt;`|
|`cm`|Centimeters (1in = 2.54cm)|`width: 5cm;`|
|`mm`|Millimeters (10mm = 1cm)|`height: 30mm;`|
|`in`|Inches|`width: 2in;`|

‚úÖ **Use When:** You want fixed dimensions regardless of screen size.

### 2. **Relative Units**
These **scale** based on the context ‚Äî like the font size of the parent or the size of the viewport. They're **more flexible** and ideal for **responsive design**.

#### üìê Font-Relative Units:

|Unit|Description|Example|
|---|---|---|
|`em`|Relative to parent element‚Äôs font size|`padding: 2em;`|
|`rem`|Relative to the **root** element‚Äôs font size|`font-size: 1.5rem;`|

- `1em` = 100% of the parent font size

- `1rem` = 100% of the root (`html`) font size

#### üì± Viewport-Relative Units:

|Unit|Description|Example|
|---|---|---|
|`vw`|1% of the **viewport width**|`width: 50vw;`|
|`vh`|1% of the **viewport height**|`height: 100vh;`|
|`vmin`|1% of the **smaller** viewport side|`font-size: 5vmin;`|
|`vmax`|1% of the **larger** viewport side|`margin: 2vmax;`|
|`%`|Relative to the **parent element‚Äôs size**|`width: 80%;`|

- `%` is **context-dependent**:
    
    - For width/height ‚Üí based on parent‚Äôs width/height
    
    - For padding/margin ‚Üí based on parent‚Äôs width
 
### Special Unit: `fr` (for Grid)

- `fr` = "fractional unit" used in **CSS Grid Layout**

```
grid-template-columns: 1fr 2fr;  /* 1 part vs 2 parts */
```

### ‚úÖ Best Practices

- Use `rem` or `em` for font sizes for better accessibility.

- Use `px` for borders, shadows, and fine control.

- Use `vw`/`vh` for responsive full-page sections.

- Avoid mixing too many unit types in one block.

## Selector in CSS
In CSS, **selectors** are patterns used to **target HTML elements** that you want to style. They tell the browser **which elements** your CSS rules should apply to.

![[Screenshot From 2025-06-19 21-54-56.png]]
### Common Types of Selectors

#### 1. Universal Selector
Selects **all** elements:

```
* {   
	margin: 0; 
}
```

#### 2. Type Selector
Selects all elements of a given tag:

```
h1 {   
	font-size: 2rem;
}
```

#### 3. Class Selector
Targets elements with a specific class:

CSS:
```
.card {   
	border: 1px solid #ccc; 
}
```

HTML:
```
<div class="card"></div>
```

#### 4. ID Selector
Targets an element with a specific ID:

CSS:
```
#main-header {   
	background-color: black; 
}
```

HTML:
```
<h1 id="main-header">Title</h1>
```

#### 5. Group Selector
Targets multiple selectors at once:

```
h1, 
h2, 
h3 {   
	font-family: Arial; 
}
```

#### 6. Descendant Selector
Targets elements **inside** other elements:

```
nav a {   
	color: green; 
}
```
<mark style="background: #FFF3A3A6;"><b>Note:</b> This selector can jump one or, multiple generation to target child, grandchild or, farther lower generation unlike <i>Child Combinator</i> which can target only direct children.</mark>
#### 7. Child Combinator (`>`)
Targets **direct children** only:

```
ul > li {   
	list-style: square; 
}
```

#### 8. Adjacent Sibling Combinator (`+`)
Selects an element **immediately after** another:

```
h1 + p {   
	margin-top: 0; 
}
```

#### 9. [Attribute Selector](https://developer.mozilla.org/en-US/docs/Web/CSS/Attribute_selectors)
Targets elements by their attributes:

```
input[type="text"] {   
	border: 1px solid #999; 
}
```

### Pseudo Class & Pseudo Element in CSS

#### [Pseudo-Classes (`:`)](https://developer.mozilla.org/en-US/docs/Web/CSS/Pseudo-classes)
A **pseudo-class** is used to define the **special state** of an element. It **doesn't add new content**, but styles the element **based on conditions** like hover, focus, or position in the DOM.

##### üìã **Syntax:**

```
selector:pseudo-class {   
	/* style rules */
}
```

##### üî∂ **Common Pseudo-Classes:**

|Pseudo-Class|Description|Example|
|---|---|---|
|`:hover`|When the user hovers over an element|`a:hover { color: red; }`|
|`:focus`|When an element (like input) is focused|`input:focus { border-color: blue; }`|
|`:checked`|When a checkbox/radio is selected|`input:checked + label { color: green; }`|
|`:nth-child(n)`|Selects the nth child|`li:nth-child(2) { font-weight: bold; }`|
|`:first-child`|Selects the first child of a parent|`p:first-child { margin-top: 0; }`|
|`:last-child`|Selects the last child of a parent|`li:last-child { color: gray; }`|
|`:not(selector)`|Excludes elements matching the selector|`p:not(.highlight) { color: black; }`|

#### [Pseudo-Elements (`::`)](https://developer.mozilla.org/en-US/docs/Web/CSS/Pseudo-elements)
A **pseudo-element** allows you to **style specific parts** of an element or **insert virtual content** before or after an element ‚Äî without needing to change the HTML.

##### üìã **Syntax:**

```
selector::pseudo-element {   
	/* style rules */ 
}
```
<mark style="background: #FFF3A3A6;"><b>Note:</b>Modern CSS uses <i>double colons (`::`)</i> to distinguish pseudo-elements from pseudo-classes, but older browsers still support single colons.</mark>

### üî∏ Common Pseudo-Elements:

|Pseudo-Element|Description|Example|
|---|---|---|
|`::before`|Inserts content **before** the element|`h1::before { content: "üëâ "; }`|
|`::after`|Inserts content **after** the element|`a::after { content: " üîó"; }`|
|`::first-letter`|Styles the **first letter** of text|`p::first-letter { font-size: 200%; }`|
|`::first-line`|Styles the **first line** of a paragraph|`p::first-line { color: navy; }`|
|`::selection`|Styles text that is **highlighted by the user**|`::selection { background: yellow; }`|

## Cascading & Specificity in CSS
### What is **Cascading** in CSS?
The **‚ÄúC‚Äù in CSS** stands for **Cascading**. It describes how **multiple CSS rules are applied in a specific order** of priority when they affect the same element.

#### üìö Cascade Determines Priority Based On:

1. **Origin**: Browser default, user styles, author styles

2. **Importance**: `!important` overrides everything (use carefully)

3. **Specificity**: How targeted the rule is

4. **Source Order**: Last rule wins if specificity is equal

### üî∂Example:

```css
/* Rule 1 */
p {
  color: blue;
}

/* Rule 2 */
#main p {
  color: red;
}

/* Rule 3 */
p {
  color: green !important;
}
```
<mark style="background: #ADCCFFA6;">Rule 3 wins because it uses `!important`, even though Rule 2 is more specific than Rule 1.</mark>

### 2. What is **Specificity**?
Specificity is a score that determines which rule wins when multiple rules apply to the same element. The **higher the specificity**, the more "weight" that selector has.

#### üßÆ Specificity Formula:
CSS calculates specificity as a **four-part value**:

$$
a (inline styles), b (id), c (.class, [attr], :pseudo-class), d (element, ::pseudo-element)
$$

#### üßæ Examples (from lowest to highest):

|Selector|Specificity|Notes|
|---|---|---|
|`p`|0,0,0,1|Element selector|
|`.text`|0,0,1,0|Class selector|
|`#main`|0,1,0,0|ID selector|
|`div#main .text`|0,1,1,1|Combination of ID + class + element|
|`style="..."`|1,0,0,0|Inline style always wins (unless `!important`)|

### `!important` Overrides Everything

If you write:

```css
p {
  color: red !important;
}
```

It will override even inline styles and highly specific selectors ‚Äî unless those also have `!important`.

==**Disclaimer:** ‚ö†Ô∏è *Use '! Important' sparingly* ‚Äì it breaks the natural cascade and can make debugging harder.==

### üß† Best Practices

- Prefer **specific classes** over IDs in CSS (more reusable and easier to override).

- Avoid relying heavily on `!important`.
 
- Understand how rules stack and resolve conflicts.

## Inheritance in CSS
**Inheritance** means that some CSS property values are **passed down** (inherited) from a parent element to its children.

Just like in families, **child elements can inherit traits (styles)** from their parents ‚Äî but **not all traits** are passed down automatically.

### üî∂ How Inheritance Works
When you apply a property to a parent element, **only certain properties** will automatically be inherited by its children ‚Äî usually those related to **text and appearance**.

### ‚úÖ Commonly Inherited Properties:

- `color`
- `font-family`
- `font-size`
- `font-style`
- `font-variant`
- `font-weight`
- `letter-spacing`
- `line-height`
- `text-align`
- `visibility`
- `white-space`
- `word-spacing`

### ‚ùå Not Inherited by Default:

- `margin`, `padding`, `border`
- `width`, `height`
- `background`, `box-shadow`
- `position`, `display`
- `flex`, `grid`, `z-index`

### üî∂Forcing Inheritance
If a property is **not inherited by default**, you can make it inherit manually:

```css
div {
  background-color: inherit;
}
```

This tells the child `<div>` to **explicitly copy** the background color from its parent.

### Example:

HTML:
```html
<div class="parent">
  Parent Text
  <p class="child">Child Text</p>
</div>
```

CSS:
```css
.parent {
  color: darkblue;
  font-family: Arial;
}

.child {
  /* Inherits both 'color' and 'font-family' */
}
```

Result: Both the parent and child text will appear in **dark blue Arial**.
